# L1-10 · Vision Transformers and CLIP

For about a decade, the conventional wisdom in computer vision was that convolutions were not optional. The inductive biases of a CNN — that nearby pixels are correlated, that features should be translation-invariant, that hierarchy is built by stacking small local filters — were considered the reason deep learning worked on images at all. Removing them sounded like removing the foundation. The Vision Transformer, published by Dosovitskiy and colleagues in 2020, removed them anyway, and worked. That result is more conceptually disruptive than it first appears: it suggests that the inductive biases of CNNs were not load-bearing assumptions about how vision must work, but a useful prior that enough data and enough compute can substitute for.

The mechanism by which ViT does this is almost embarrassingly direct. An image is cut into a grid of fixed-size patches — typically 16×16 pixels. Each patch is flattened into a vector and linearly projected into an embedding. You now have a sequence of patch embeddings, which is exactly the kind of input a standard transformer encoder expects. Add positional encodings so the model knows which patch came from where, prepend a learnable classification token, and run the whole thing through the same self-attention stack you would use for language. The transformer does not know it is looking at an image. It is performing the same query-key-value operation it performs on text tokens, except now each "token" is a 16×16 patch of pixels.

The cost of throwing away the convolutional prior is that ViT is hungry. Without the built-in assumption that nearby pixels matter more than distant ones, the model has to learn that fact from data. On ImageNet alone, a CNN of comparable size will outperform a ViT. It is only when ViT is pre-trained on much larger datasets — JFT-300M, for instance — that it begins to match and then exceed CNN performance. This is the recurring tradeoff of the transformer family: weaker inductive bias, higher data requirement, higher ceiling. The same pattern shows up in language, where transformers needed pre-training corpora at a scale that earlier architectures never required.

CLIP, which Radford and colleagues at OpenAI published in 2021, took the next conceptual step and is in some ways the more important model. The premise was disarmingly simple: instead of training an image model to predict ImageNet's thousand class labels, train it to predict which caption goes with which image. The dataset was 400 million image-caption pairs scraped from the web. The architecture was two encoders — one for images (a ViT or a CNN), one for text (a transformer) — that produce vectors in a shared embedding space. The training objective was contrastive: in a batch of, say, 32,000 image-text pairs, push each image's embedding toward its true caption's embedding and away from all the others. The result is a joint space where semantically related images and texts land near each other.

What this buys you is something the field had been chasing for years and had never quite achieved: zero-shot transfer. To classify an image as "a photo of a dog" with CLIP, you embed the image, embed the candidate text strings, and pick the closest one. No retraining. No fine-tuning. CLIP can classify images into categories it was never explicitly taught, because the text encoder gives you a free interface to any concept describable in language. This was the empirical result that made the term "foundation model" feel earned rather than aspirational — a single pre-trained system that could be redirected at new tasks with no parameter updates at all.

The reason CLIP matters specifically for generative AI is that it solved the conditioning problem for text-to-image systems. When you type a prompt into Stable Diffusion or DALL-E 2, the prompt does not enter the diffusion model directly; it is first encoded by CLIP's text encoder into a vector that lives in the shared image-text space. The diffusion model has been trained to take that vector as a conditioning signal and produce an image whose own embedding would land near it. CLIP is the bridge between language and pixels. Without it, or something like it, prompt-following would not work — and the failure modes of CLIP (its vocabulary gaps, its biases, its difficulty with compositional prompts like "a red cube on top of a blue sphere") propagate directly into the failure modes of every text-to-image system built on top of it.

The skill this topic builds is the ability to read a multimodal architecture diagram and know what each box is doing and why. When you see a Stable Diffusion pipeline diagram with a CLIP text encoder feeding into a U-Net feeding into a VAE decoder, the CLIP component should not feel like a black box labelled "text goes in." It should feel like a specific, comprehensible mechanism: a contrastively-trained encoder that maps language into a space where image embeddings also live, allowing the diffusion model to be steered by language without ever having been trained on language directly. That clarity — the ability to trace a prompt through the system and predict where it will fail — is the difference between using these tools and reasoning about them.

## Level 2 candidates

**Patch embedding in ViT** — The conversion of an image into a sequence of flattened patch vectors, including the choice of patch size, the linear projection, and the role of the classification token. Worth depth because the implications for computational cost (attention is quadratic in sequence length, and patch size sets that length) and inductive bias relative to CNNs are the substantive reason ViT requires more data to train.

**Contrastive learning objective** — The InfoNCE-style loss that makes paired image-text embeddings similar and unpaired ones dissimilar, including the role of batch size, temperature, and the symmetric formulation across both modalities. Worth depth because contrastive learning is now a general-purpose technique beyond CLIP, and understanding why it works requires engaging with what the negative samples in a batch are actually teaching the model.

**CLIP as a text encoder for diffusion** — The specific path by which a text prompt enters a Stable Diffusion pipeline, what cross-attention does with the CLIP embedding, and how the choice of CLIP model (e.g. OpenCLIP variants) shapes generation. Worth depth because most practical prompt engineering decisions become legible only once you understand what the conditioning signal actually is and where it intervenes.

**Zero-shot transfer from CLIP** — The mechanism by which CLIP is repurposed for classification, retrieval, and generation conditioning without task-specific training, and the limits of that transfer. Worth depth because zero-shot transfer is the empirical foundation of the foundation-model paradigm, and understanding its boundaries (where CLIP's vocabulary fails, where its biases surface) is a transferable skill across modern multimodal systems.

**ViT vs CNN inductive biases and data requirements** — The tradeoff between weaker priors and higher data needs, and the empirical scaling regime where ViT begins to dominate. Worth depth because this same tradeoff structures every architectural decision in modern deep learning, not just vision, and seeing it concretely in the ViT-vs-CNN comparison sharpens the general intuition.
